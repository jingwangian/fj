What's New in FUJITSU Enterprise Postgres 9.5?

Lightning-fast performance with In-Memory Column Store

The In-Memory Column Store (IMCS) engine, developed by Fujitsu Laboratories Ltd, is integrated with FUJITSU Enterprise Postgres, and provides significantly faster analytical query processing by storing a columnar representation of row-oriented data in memory. On a single server with 15 CPU cores, performance is at least 50 times faster than before.

IMCS instantly reflects updated row-oriented data to its columnar equivalent, stores it without being dependent on memory capacity, and quickly conducts analysis of that data. Massive volumes of column-oriented data can be stored by taking advantage of this new technique for managing data.

Data masking for increased data security

With the new data masking feature, sensitive data such as credit card numbers, bank account details and other sensitive personal information can be masked. Different privileges-different views-the same data-enables employees in a bank call center, for example, to see a restricted view of sensitive data, whereas employees in a bank branch can see a full view of all the customer information.

FUJITSU Enterprise Postgres uses masking policies to obscure data using a variety of masking methods, including: character shuffling, nulling or deletion, encryption, masking, and word substitution.

Data masking makes it safe to use production data in a testing or development environment.

Simplified operation with WebAdmin

FUJITSU Enterprise Postgres provides an enhanced WebAdmin cluster management console. The WebAdmin GUI has been redesigned to improve the user experience with a simpler, more intuitive design for tasks such as FUJITSU Enterprise Postgres setup, database backups and recovery.

New features include:

  • Centralized administration of the database instances running on multiple server machines
  • Comprehensive and easy-to-use credential management
  • Replication functionality in various configuration modes

Parallel scanning and aggregation for faster query processing

The new parallel scanning and aggregation features also provide significant performance improvements‐query performance can be exponentially improved based on the number of cores by distributing the query execution process over multiple CPU cores.

Parallel scanning and aggregation are execution features that the query planner can automatically use. They use available resources within the server to speed up scanning and aggregation.

PostgreSQL 9.5 features

FUJITSU Enterprise Postgres includes the new features in PostgreSQL 9.5, which was released in January 2016:

  • Row level security

    • This feature enables security policies to be set to restrict, filter or control access to the data.

  • UPSERT

    • This feature has been requested by many application developers for years, and PostgreSQL 9.5 delivers. It enables the database to handle conflicts between concurrent data changes. This allows legacy MySQL applications to be migrated to PostgreSQL.

  • Big Data features

    • PostgreSQL 9.5 includes multiple Big Data features such as faster sorts, foreign data wrappers (FDWs), TABLESAMPLE, CUBE, ROLLUP, GROUPING SETS, and BRIN indexing.

Refer to http://www.postgresql.org/about/news/1636/ for details on all that's new in PostgreSQL 9.5.
